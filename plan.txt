4.2 Layer Selection Strategy
Prior work (Panickssery et al., 2024; Zou et al., 2023) shows that middle layers (roughly 40-65% of total depth) are most steerable. For a 32-layer model like Sarvam-1, this means layers 13-21. The protocol is:
5.	Run a small probe study on 50 pairs per category to measure the contrastive activation magnitude per layer.
6.	Select the 3 layers with highest magnitude separation between safe and unsafe activations.
7.	Extract full vectors only from these 3 layers to reduce compute. Use the layer with highest separation as your primary intervention point.

4.3 Extraction Code (Step-by-Step)
The following is the complete extraction pipeline. Save as extract_vectors.py:
import torch, json, numpy as np
from transformers import AutoTokenizer, AutoModelForCausalLM
from transformer_lens import HookedTransformer

MODEL_ID = 'sarvamai/sarvam-1'
model = HookedTransformer.from_pretrained(MODEL_ID, dtype=torch.float16)
tokenizer = AutoTokenizer.from_pretrained(MODEL_ID)

def get_residual_stream(prompt, layer_idx):
    tokens = tokenizer(prompt, return_tensors='pt').input_ids.cuda()
    _, cache = model.run_with_cache(tokens)
    return cache[f'blocks.{layer_idx}.hook_resid_post'][0, -1, :].float()

def extract_steering_vector(pairs, layer_idx, category, lang):
    safe_acts, unsafe_acts = [], []
    for pair in pairs:
        prompt_safe = pair['prompt'] + ' ' + pair['safe']
        prompt_unsafe = pair['prompt'] + ' ' + pair['unsafe']
        safe_acts.append(get_residual_stream(prompt_safe, layer_idx))
        unsafe_acts.append(get_residual_stream(prompt_unsafe, layer_idx))
    vector = torch.stack(unsafe_acts).mean(0) - torch.stack(safe_acts).mean(0)
    return vector / vector.norm()  # L2-normalise

Run extraction on Azure ML Compute Cluster (Standard_NC6s_v3). Log each vector artifact to the Azure ML model registry under 'steering-vectors/{model}/{lang}/{category}/layer{N}'.

5. Inference-Time Intervention
5.1 Architecture
The inference pipeline has three components that run in sequence within a single forward pass:
8.	Language & Risk Classifier (IndicBERT): Detects the input language and predicts the most probable harm category. Runs in under 10ms on CPU.
9.	Vector Selector: Looks up the pre-computed steering vector for (model, language, category, layer).
10.	Residual Stream Hook: Adds alpha * steering_vector to the residual stream at the target layer during the model's forward pass.

5.2 IndicBERT Classifier Setup
Use ai4bharat/indic-bert from HuggingFace. Fine-tune on your dataset's prompt-only inputs (no completions) for an 8-way category classification task. This is a lightweight fine-tune (only the classification head) that takes under 30 minutes on a V100.

5.3 The Steering Hook
def make_hook(steering_vector, alpha=15.0):
    def hook_fn(value, hook):
        return value + alpha * steering_vector.to(value.device)
    return hook_fn

def steered_generate(model, tokenizer, prompt, vector, alpha, layer_idx, max_new=200):
    hook = make_hook(vector, alpha)
    with model.hooks(fwd_hooks=[(f'blocks.{layer_idx}.hook_resid_post', hook)]):
        tokens = tokenizer(prompt, return_tensors='pt').input_ids.cuda()
        out = model.generate(tokens, max_new_tokens=max_new)
    return tokenizer.decode(out[0], skip_special_tokens=True)

5.4 Alpha Calibration
Alpha controls the steering strength. Too low = no effect. Too high = incoherent outputs. Calibrate per category using your validation split. For each category, sweep alpha in [5, 10, 15, 20, 25] and compute:
•	Safety Rate: % of outputs classified as safe by Azure AI Content Safety API
•	Fluency Score: BLEURT or perplexity relative to unsteered output
•	Optimal alpha: highest safety rate with perplexity increase < 20%
Typical values from prior work: alpha 12-20 for most harm categories.

6.2 Key Metrics
•	Safety Improvement Rate (SIR): (# safe steered outputs - # safe unsteered outputs) / total prompts. Target: > 70% for primary categories.
•	False Positive Rate (FPR): % of benign prompts incorrectly classified as harmful and steered unnecessarily. Target: < 10%.
•	Fluency Preservation: Average BLEURT score of steered vs. unsteered on benign inputs. Target: > 0.85.
•	Latency Overhead: Mean additional time per token from the hook. Target: < 3ms.
